2.5 Implementation & Design Choices

• LLM selection: Groq Llama 3.3 70B (or equivalent) chosen for high-quality scenario generation
and low latency; prompt templates include explicit safety and factuality constraints.


• RAG pipeline:All DG responsestmust include source chunks (citations) to ensure factual
grounding and educational integri y.
• Personalization: ELO + streak adjustments form fast converging difﬁculty mapping; long-term
personalization uses category-wise proﬁciency stats.


• Privacy: By default, store only non-sensitive aggregate metrics; optional advanced telemetry is
opt-in and processed on device where feasible.


• Explainability: Feedback contains three levels — Quick Tip, Educated Explanation, and Deep
Dive — to suit novice→expert learners.


• Scalability: Use caching for repeated RAG queries and scenario templates; vector DB indexed
for low-latency retrieval.



2.6 Success Criteria & Evaluation Plan

•	System metrics: Scenario gen time < 3s; DG response < 2s; retrieval latency < 0.5s.
•	Learning metrics: Pre/post accuracy improvement > 20%; retention test >70% after one
week.


•	Engagement metrics: DAU > 60% within pilot cohort; completion rate > 75% for 10+
scenarios.


•	Qualitative: User satisfaction > 4.0/5 for scenario realism and DG helpfulness.
